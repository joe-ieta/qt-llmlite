# v0.2.1

本次版本的重点不是发布一个完整的翻译 App，而是验证当前 `qt-llm` 底层库对业务智能体开发的支撑能力，尤其是以下几个方面：

- 是否能够在 `src/apps` 之外，以独立目录构建一个业务 Agent。
- 是否能够复用当前底层库的大模型接入能力，并支持不同 skill 绑定不同模型。
- 是否能够把 MCP 能力接入到业务 Agent 的工作流里，而不是只停留在基础工具层。
- 是否能够支撑一个具备片段、多文档、批量后台任务等典型 Agent 工作模式的产品骨架。

围绕这个目标，本次完成了一个独立的 `pdf_translator_agent` 验证工程，并落下了以下关键能力：

- 独立业务 Agent 工程结构与主程序入口
- Skill 框架与运行时注册机制
- Skill 级模型路由
- MCP 接入骨架与文档解析 / 术语库能力验证
- 片段翻译、单文档翻译、批量后台翻译三种基本工作模式
- 对照阅读、任务状态、批量队列、配置持久化等产品级骨架能力

这意味着当前仓库已经基本验证了这样一条路线：

可以在不污染基础工具目录的前提下，基于 `qt-llm` 底座快速搭建业务智能体，并把 LLM、MCP、任务状态机、界面交互和文件持久化统一到一个独立应用中。

当前仍未作为本次版本目标闭环的内容包括：

- 页面级 PDF 阅读器体验
- 独立 OCR skill 的完整交付
- Linux 端构建与运行验证
- 面向最终交付的完整翻译产品打磨

因此，`v0.2.1` 更适合作为一次 Agent 开发支撑能力验证版，而不是翻译产品正式版。
