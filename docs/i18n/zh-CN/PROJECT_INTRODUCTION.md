# 项目介绍

`qt-llm` 是一个轻量化的、基于 Qt 的 LLM 界面层工程，用于为桌面应用提供简洁实用的 Qt/C++ 能力底座，并内置支持：

- LLM 会话
- Tools
- MCP
- SKILL
- Agent 工作流

该工程的目标是让开发者能够以代码级方式，直接把这些能力集成到自己的 Qt 工程中，同时保持核心基础层、实用工具层和业务 Agent 层的清晰分离。

## 项目定位

这个工程不只是一个聊天示例，也不只是一个模型 Provider 封装层。

它的核心目标是提供一个轻量化、面向 Qt 的 LLM 界面层，使开发者可以：

- 以统一接口接入多个 LLM 后端
- 构建多轮会话流程
- 构建工具调用流程
- 接入 MCP Server 和 MCP 工具能力
- 在产品中组织 SKILL 能力单元
- 在同一套底座上构建面向任务流的 Agent

## 顶层三分三层

当前仓库采用三层顶层结构。

### 1. `qtllm`（`src/qtllm/`）

这是核心层。

它提供可复用的基础能力，包括：

- Provider 抽象
- 请求与响应执行
- 流式响应处理
- 会话管理
- Tool Runtime
- MCP 集成
- 存储
- 日志与链路追踪
- 运行过程可观测性支撑

### 2. `apps`（`src/apps/`）

这是实用工具层。

它承载基于核心层构建的宿主程序和工具程序，例如：

- 聊天宿主示例
- MCP 管理工具
- `toolsinside` 监控追踪与分析工具
- Tool 与 Workspace 管理工具

这些 Apps 既是可直接使用的实用工具，也是基于 `qtllm` 构建应用的参考实现。

### 3. `agents`（`src/agents/`）

这是业务 Agent 层。

它用于构建一些常用 Agent，并作为 Agent 开发的参考实现。这里的程序不再是基础设施演示，而是验证如何在共享底座之上构建面向工作流的业务应用。

当前的参考 Agent 为：

- `src/agents/pdf_translator_agent/`

## 集成方式

该项目强调以代码级方式集成到用户自己的 Qt 工程中。

推荐使用方式是：

- 在 Qt 工程中直接链接 `qtllm` 库
- 在 C++ 代码中配置模型 Provider 和运行时能力
- 在自己的界面和业务逻辑中组合会话、Tool、MCP、SKILL 和 Agent 流程

设计目标是让 Qt 开发者能够以简洁直接的方式接入，而不是先强制引入额外的服务化架构。

## 运行过程监控、追踪与分析

该项目的重点不只是在“调用模型”。

它同时强调对以下运行过程的监控、追踪和分析支持：

- LLM 会话过程
- Tool 执行过程
- MCP 调用过程
- SKILL 执行路径
- Agent 工作流阶段

当前仓库已经具备或正在验证以下支撑能力：

- 结构化日志
- 运行链路追踪
- Artifact 记录
- 工作流状态检查
- 以 `toolsinside` 为代表的分析工具支持

因此，这个工程不仅适合“做出一个 LLM 功能”，也适合持续观察、调试和优化这些能力在真实应用中的运行情况。

## 典型使用场景

典型使用场景包括：

- 在 Qt 桌面应用中嵌入 LLM 聊天能力
- 为现有 Qt 产品增加 Tool 调用和 MCP 能力
- 围绕 LLM 工作流构建内部运营或研发工具
- 构建带任务流、持久化状态和界面的领域 Agent

## 环境与技术栈

- 语言：C++17
- 框架：Qt
- 构建基线：qmake
- 目标形态：轻量化桌面集成
- 当前主要验证环境：Windows
- 目标运行平台：Windows 和 Linux

该项目以 Qt 作为应用与界面的主要基础，而不是围绕 Web First 技术栈做包装。

## 当前仓库方向

在当前阶段，这个仓库应理解为：

- 一个可复用的 Qt/C++ LLM Core
- 一组实用的宿主工具和辅助工具
- 一个持续扩展的业务 Agent 参考区

通过这样的结构，可以把基础设施、工具程序和 Agent 产品清晰分离，同时共享同一套 LLM、Tools、MCP、SKILL 和可观测性能力底座。
